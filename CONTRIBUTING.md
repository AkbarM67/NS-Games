# Contributing Guidelines

Thank you for your interest in contributing to NS Topup Games Platform.

## Development Setup

1. Fork the repository
2. Clone your fork locally
3. Follow the setup instructions in README.md
4. Create a feature branch

## Code Standards

### Backend (Laravel)
- Follow PSR-12 coding standards
- Use meaningful variable and function names
- Add proper documentation for API endpoints
- Write tests for new features

### Frontend (React)
- Use TypeScript for type safety
- Follow React best practices
- Use consistent component structure
- Implement proper error handling

## Commit Guidelines

Use conventional commit format:
```
type(scope): description

feat: add new payment method
fix: resolve authentication issue
docs: update API documentation
```

## Pull Request Process

1. Ensure all tests pass
2. Update documentation if needed
3. Add changelog entry
4. Request review from maintainers

## Testing

- Run backend tests: `php artisan test`
- Run frontend tests: `npm test`
- Test API endpoints manually
- Verify UI functionality

## Questions?

Contact the development team for any questions or clarifications.
